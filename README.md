## What is Nova OS?

Nova sits between your AI agents and the real world.

Every action an agent wants to execute passes through Nova first. Nova evaluates it in under 5ms, logs it to an immutable ledger, and either approves, blocks, or escalates ‚ÄĒ before anything reaches production.

```
agent wants to act  ‚Üí  Nova evaluates (<5ms)  ‚Üí  APPROVED / BLOCKED / ESCALATED  ‚Üí  world
```

Not another agent. Not another chatbot. **Infrastructure.**

---

## The problem

AI agents execute actions without control. They hallucinate endpoints, send duplicate emails, write to wrong databases, run destructive commands ‚ÄĒ and nothing stops them.

When something goes wrong: no trace. No brake. No rollback.

Nova fixes that.

## How it works

```
Score ‚Č• 70   ‚Üí  ‚úď  APPROVED   ‚ÄĒ executes immediately
Score 40-70  ‚Üí  ‚ö†  ESCALATED  ‚ÄĒ pauses, waits for your decision
Score < 40   ‚Üí  ‚úó  BLOCKED    ‚ÄĒ stopped, logged, explained
```

Every decision is written to the Intent Ledger. Cryptographic. Auditable. Permanent.

## nova guard ‚ÄĒ one command, all agents

```bash
nova guard
```

Nova scans your environment, detects every AI agent running, and puts all of them under governance. No config files. No manual setup per agent.

```bash
nova guard --path .env      # protect a specific path from ALL agents
nova guard --path /prod     # nothing in /prod can be touched
```

## nova run ‚ÄĒ wrap any command

```bash
nova run "pm2 restart melissa"
nova run "rm -rf /tmp/old-logs"
nova run "git push origin main --force"
```

Nova classifies the risk, shows you the verdict, and either executes or blocks. Every run is logged.

## nova shield ‚ÄĒ HTTP proxy

```bash
nova shield
```

Starts a proxy on `127.0.0.1:7755`. Every HTTP request from your agents passes through it. Nova validates the action before forwarding. Nothing reaches your services unexamined.

## Governance flow

```
1. DISCOVER    ‚Üí  auto-detect agents on your machine
2. CONNECT     ‚Üí  attach rules: cannot_do, must_confirm, rate_limits
3. INTERCEPT   ‚Üí  every action passes through Nova before execution
4. EVALUATE    ‚Üí  deterministic rules + optional LLM validation in <5ms
5. DECIDE      ‚Üí  APPROVED / BLOCKED / ESCALATED / DUPLICATE
6. LEDGER      ‚Üí  cryptographic, immutable, permanent record
```

## Architecture

```
nova.py              CLI entrypoint + local server launcher
nova/                core: API, kernel, ledger, discovery, storage
frontend/            React dashboard at localhost:9800
n8n-nodes-nova/      n8n native integration
legacy/              compatibility layer ‚ÄĒ being cleaned up
docs/                deployment, API reference, architecture
tests/               platform, API, discovery, runtime tests
```

**Core design decisions:**

- **Fail-open** ‚ÄĒ Nova never blocks your work if it goes down
- **LLM-optional** ‚ÄĒ 90% of validations run without any AI call
- **Hot-reload rules** ‚ÄĒ update governance policies without restart
- **Immutable ledger** ‚ÄĒ cryptographic chain, nothing can be deleted
- **Transparent proxy** ‚ÄĒ wraps agents from outside, zero modification to agent code
- **Offline mode** ‚ÄĒ actions queue locally and sync when server returns
- **Bilingual** ‚ÄĒ ES/EN native, no config required
